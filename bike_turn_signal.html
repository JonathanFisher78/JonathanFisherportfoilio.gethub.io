Bike Turn Signal

The Problem

I'm a college student at Georgia Tech, and I commute to campus every day on my bike. Navigating busy streets with lots of traffic and students unfamiliar with bike signals can be challenging. Signaling turns is particularly difficult as both hands are needed on the handlebars, especially when coming to a stop.

The Proposal

My solution is to design turn signals for my bike using LEDs and an Arduino microcontroller. The design includes left, right, and brake lights located on the back of the bike. A magnetic reed switch attached to the brakes triggers the brake lights. When the brake is engaged, disconnecting the magnet, the brake lights illuminate. Additionally, a quick pull and release of the brake signal a turn, either left or right. With this project I hope to learn soldering, how to use reed switches, as well as further my understanding of Arduino and CAD.

The Process

Although fulling building the device is outside the scope of this project, I initially constructed the circuit on a breadboard using reed switches, conducting extensive code debugging, and testing. In the next step I designed a housing in CAD using Autodesk Inventor to accommodate the battery, Arduino, and LED arrangement.

The Product

circuit is fairly simple, consisting of just LEDs, a reed switch, resistors, and the Arduino. The majority of this project was code-based. I started with code to learn how the reed switches work, and once I understood, I moved on to a [Debounce code].


int buttonPin1 = 12;    // replace with your button pin
int ledPin1 = 9;      // replace with your LED pin

int buttonPin2 = 13;    // replace with your button pin
int ledPin2 = 10;      // replace with your LED pin

int debounceDelay = 50;

// variables for debounce
int buttonState = LOW;              // current state of the button
int lastButtonState = LOW;          // previous state of the button
unsigned long lastDebounceTime = 0; // last time the button state was toggled

void setup() {
    Serial.begin(9600);
    pinMode(buttonPin1, INPUT);
    pinMode(ledPin1, OUTPUT);

    // set initial LED state
    digitalWrite(ledPin1, LOW);
}

void loop() {
    checkButton(buttonPin1, ledPin1, debounceDelay);

}

void checkButton(int buttonPin, int ledPin, int debounceDelay) {
    int reading = digitalRead(buttonPin);

    if (reading != lastButtonState) {
    lastDebounceTime = millis();
    Serial.println(lastDebounceTime);
    }

    if ((millis() - lastDebounceTime) > debounceDelay) {
    //Serial.println("its time");

    if (reading != buttonState) {
        Serial.println("button changed");
        
        buttonState = reading;
        if (buttonState == HIGH) {
        Serial.println("Button pressed!");
        digitalWrite(ledPin, HIGH);
        delay(300);
        digitalWrite(ledPin, LOW);
        delay(300);
        }
    }
    }

    lastButtonState = reading;
}"
The point of the Debounce code was to detect if the reed switch had been pulled for a bit of time and then replaced. Without it, when the bike goes over a bump or rough ground, it may accidentally trigger the signal when I do not want it to.

Next, I made a function for blinking the signal.

int leftArrow = 9;       // replace with your left arrow LED pin
int rightArrow = 10;     // replace with your right arrow LED pin

void setup() {
    pinMode(leftArrow, OUTPUT);
    pinMode(rightArrow, OUTPUT);

    // set initial LED states
    
    digitalWrite(leftArrow, LOW);
    digitalWrite(rightArrow, LOW);
}

void loop() {
    // Example usage:
    blinkArrow("left");
    delay(1000);  // Wait for 1 second
    blinkArrow("right");
    delay(1000);  // Wait for 1 second
}

void blinkArrow(const char* direction) {
    if (strcmp(direction, "left") == 0) {
    // Blink left arrow 3 times
    for (int i = 0; i < 3; ++i) {
        digitalWrite(leftArrow, HIGH);
        delay(300);
        digitalWrite(leftArrow, LOW);
        delay(300);
    }
    // Ensure right arrow is off
    digitalWrite(rightArrow, LOW);
    } else if (strcmp(direction, "right") == 0) {
    // Blink right arrow 3 times
    for (int i = 0; i < 3; ++i) {
        digitalWrite(rightArrow, HIGH);
        delay(300);
        digitalWrite(rightArrow, LOW);
        delay(300);
    }
    // Ensure left arrow is off
    digitalWrite(leftArrow, LOW);
    } else {
    // Invalid direction
    Serial.println("Invalid direction");
    }
}
It takes a direction, "left" or "right," and flashed the appropriate arrow three times. After this, I combined it all together into a complete code [Download Here], adding in the brake light functionality. Below is a video of the prototype in action.

The Possible

Although I did not get to build this project completely, I plan in the future printing my housing and wiring it all up to my bike. Future improvements for this project include adding front lights to the bike for head-on signaling. Considerations are also being made to implement a separate button for signaling, as relying on the brake alone might not be convenient. Additionally, efforts will be made to enhance the bike-mounted components' security, exploring alternatives to the current use of tape for attachment.
